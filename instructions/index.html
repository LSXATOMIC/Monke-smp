Make a Minecraft map

Collaboratively create maps of the server you play on without without needing admin or hosting

View project onGitHub
Example map

The map lets any player show locations of interest without spoiling the element of exploration.

(Alternately, for anyone with admin access to the server, their own hosting, and who doesn't mind removing the element of exploration from the game, Overviewer is a nice online map system)

Creating your own

  1. Create a file in Google Drive and place all your map locations into it.
  2. Set the file's visibility to "Public on the web" and add its id to the end of this URL:
    https://googledrive.com/host/0B35KCzsTLKY1bXBnOGdSR0V0ZUU/index.html?googlesrc=
  3. You're done!

In more detail...

To create your own map you take the web-address of the map page and add your own settings to the end of the URL. One of the settings you add is the web-address of a page containing all the locations you want displayed on your map.

If you're going to collaboratively build the map with others, then a shared editable file on Google Drive or a wiki page are good places to keep the list of locations - everyone can edit the list and the map page can read the locations directly from it. The locations list can be in either html format or txt format.

If you want to host it yourself, then download the zip file on the right, otherwise just use the one already hosted on Google Drive. For security reasons, browsers won't allow the map page to access your locations list unless both their addresses share the same domain, so...

this means if you don't want to host the map page yourself, and want to use the one already on Google Drive, then you must also create you locations list on Google Drive.

Location list format

The Location list file is a list of locations, one per line. Each line gives a location type and its Minecraft x and z coordinates. You can use F3 in Minecraft to get coordinates.

If your location list file is a text file, then locations are specified in comma separated format, for example:

type,             x,     z,    description,                      owner,       href, icon-index

DesertVillage,  245,   350, "Ghost town" 	
WitchHut,     -4056, -2487,
PlayerHouse,   -800, -1800, "RedstoneRising's mountain lair", "RedstoneRising", "", 21

If the location list is an html file (for example a wiki page), then the locations can be specified either with unordered "bullet" lists, e.g:

  • DesertVillage, 245, 350, "Ghost town"
  • WitchHut, -2056, -2487
  • PlayerHouse, -800, -1800, "RedstoneRising's mountain lair", "RedstoneRising", "", 21

or by using a table:

Type x z DescriptionOwnerhrefIconIndex
DesertVillage 1245 350 Ghost town
WitchHut -2056 -2487
PlayerHouse -400 -1800 RedstoneRising's mountain lairRedstoneRising 21

Or with a combination of bullet lists and tables. Because of the examples above, even this page would work as a location list!

Settings for the map, such as scale can also be specified in the location list file.

Locations

Each location may have up to 7 columns of information specified:
type, x, z, description, owner, href, icon-index
but only the first 3 items are required (type, x, z).

type

The first column in the table specifies the type of location, and must be one of the following:

  • Spawn
  • PlayerHouse
  • PlayerCastle
  • PlayerStructure
  • EnchantingRoom
  • Village
  • DesertVillage
  • JungleTemple
  • DesertTemple
  • WitchHut
  • NetherFortress
  • NetherPortal
  • Forest
  • MushroomIsland
  • Horse
  • Wolf
  • Dragon
  • Label

The location-type determines the default icon to be drawn on the map at the location, and some location-types also provide an href, (for example, Village locations will link to "http://minecraft.gamepedia.com/Village by default). However both the icon and the href from the location can be overridden by putting your own values in the appropriate column.

The location-type also makes the list easier to read.
